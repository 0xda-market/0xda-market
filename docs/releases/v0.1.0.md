This is the first versioned release of the provider-agnostic 0xda Market core.

## Highlights

- Durable intent, quote and order execution with idempotent PostgreSQL storage.
- Authenticated public/operator APIs and provider-independent Telegram users.
- Database-owned product catalog with localized `en_US` and `uk_UA` names.
- Auditable USDT pricing with a current snapshot and append-only history.
- Separate test (`master`) and production (`release`) Render environments.

## Compatibility note

**Breaking:** `premium_9m` is replaced by `premium_12m`. Migration `006`
preserves the product position and rewrites existing price-history references.
Consumers must use `premium_12m` after upgrading.

Deploy this release together with
[`0xda-market-bot` v0.1.0](https://github.com/0x0sky/0xda-market-bot/releases/tag/v0.1.0).
The core must be promoted and verified before the bot.

## Database migrations

| Migration | Purpose |
| --- | --- |
| `006_replace_premium_9m_with_12m` | Replace the SKU while preserving price history. |
| `007_product_catalog_localizations` | Add localized catalog copy, current-price snapshots and internal editor UUIDs. |
| `008_legacy_catalog_rollback_window` | Retain deprecated catalog columns for one application-rollback window. |

The container applies migrations transactionally before Puma starts. Take a
database backup before promotion. Database migrations are forward-only; use a
revert/forward-fix deployment rather than manually deleting migration records.

## Verification after promotion

- `GET /health` returns HTTP `200` and reports a healthy database.
- `GET /v1/products?locale=en_US` and `?locale=uk_UA` each return nine products.
- The catalog contains `premium_12m` and does not contain `premium_9m`.
- An admin price update records an internal editor UUID and updates the current
  product price snapshot.
